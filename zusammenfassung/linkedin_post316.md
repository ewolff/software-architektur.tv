# 🤖 KI macht Software billiger – aber macht sie Projekte wirklich einfacher?

Eine wichtige Diskussion vom #TechRidersSummit: Sebastian Kleinschmager (SparePartsNow) und Axel Schulz stellen die kritischen Fragen, die wir uns stellen müssen.

**Das Versprechen:** KI beschleunigt die Codeerstellung deutlich! 🚀

**Die Realität:** Schnellerer Code ≠ besserer Code. Und schneller Code ≠ einfachere Projekte.

## Was wir gelernt haben:

✅ **Coding wird schneller**
❌ **Wartung wird teurer** – oft nichjt mitgedacht
⚠️ **Teamarbeit bleibt komplex** – hier helfen KI-Prompts wenig

## Die unbeantworteten Fragen:

- Wer haftet für KI-generierten Code mit Security-Lücken?
- Entwickeln wir gerade zu viel Software, die niemand mehr warten kann?
- Kommt die Spec-Driven-Development-Era?
- Welches technisches Know-how brauchen Softwareentwickler:innen in 5 Jahren?

## Der Knackpunkt:

Die Diskussion zeigt: **Softwareentwicklung ist mehr als Codinggeschwindigkeit.** Es geht um Architektur, Wartbarkeit, Sicherheit – und um Menschen, die verstehen, was sie tun.
